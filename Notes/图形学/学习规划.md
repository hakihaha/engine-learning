# 图形学与引擎开发一年学习规划（详细版）

> 本学习规划覆盖从零基础到能够独立完成一个 mini 渲染器或 mini 引擎的完整路径，并附带学习资源（含 BattleFire 推荐购买课程）、分阶段任务、每阶段输出内容。
> 
> 适合方向：图形学 / 引擎开发（C++）。

---

## 🌕 总体学习周期：52 周（1 年）

每周投入：**6–10 小时**（可调）

核心目标：

1. C++ 能达到熟练应用（现代 C++）
    
2. 理解 GPU 渲染管线（OpenGL → D3D12）
    
3. 能写 Shader、理解光照模型
    
4. 能理解并实现基本渲染器功能
    
5. 能构建作品集（mini-engine / renderer）
    

---

# 🧭 第一阶段（第 1–8 周）——基础能力打牢

## 🎯 阶段目标

- 建立扎实 C++ 基础
    
- 掌握三维数学与矩阵推导
    
- 为 OpenGL / D3D12 做好准备
    

## 📘 要学内容

### 1. C++ 强化（1–4 周）

重点：

- 指针、引用、内存模型
    
- 类与继承、虚函数
    
- 模板、STL、迭代器
    
- 现代 C++：智能指针、lambda、move 语义
    

**推荐资源：**

- 《C++ Primer》第 5 版
    
- B 站：侯捷 C++ 系列
    
- Udemy: Modern C++ courses（英文可选）
    

### 2. 图形学数学（5–8 周）

重点：

- 线性代数（向量、矩阵、叉乘、点乘）
    
- 模型矩阵 / 视图矩阵 / 投影矩阵推导
    
- 左手坐标系与右手坐标系
    

**BattleFire 课程：必买**

- 左手坐标系数学
    
- 右手坐标系数学
    

这两个是图形学核心基础。

---

# 🧭 第二阶段（第 9–24 周）——OpenGL（主战场）

> OpenGL 用来建立**渲染体系完整理解**，比 D3D12 适合入门。

## 🎯 阶段目标

- 输出完整 OpenGL 渲染管线理解
    
- 能做简单渲染 demo
    
- 能写 Shader
    

## 📘 需要 BattleFire 课程

**必买：**

- OpenGL 入门
    
- OpenGL 进阶
    
- OpenGL Shader
    

---

## 📅 详细任务安排

### 第 9–12 周：OpenGL 入门

内容：

- GLFW + glad 环境搭建
    
- VAO / VBO
    
- 上屏三角形
    
- Shader 基础（vs/fs）
    
- 纹理加载
    

输出：

- 上屏 demo
    
- 简单纹理渲染
    

### 第 13–18 周：OpenGL 进阶

内容：

- 摄像机系统
    
- 深度测试、混合
    
- 模型加载（Assimp）
    
- 阴影（Shadow Map）
    
- FBO 后处理（Bloom、HDR）
    

输出：

- 一个简单的场景渲染 demo
    

### 第 19–24 周：OpenGL Shader 高阶

内容：

- 光照（Phong、Blinn）
    
- 基于物理的渲染（PBR）基础
    
- BRDF 基础
    
- GPU 计算（了解 Compute Shader）
    

输出：

- 带光照的渲染 demo（含 PBR）
    

---

# 🧭 第三阶段（第 25–36 周）——Direct3D 12（你已购课程）

> D3D12 更贴近现代图形 API，复杂但学习价值极高。

## 🎯 阶段目标

- 理解底层图形 API
    
- 能用 D3D12 实现基础渲染
    
- 理解 descriptor heap、pipeline state
    

## 📘 BattleFire 课程

**你已经买：**

- Direct3D 12 萌谷手册（够用了）
    

## 📅 周任务

### 第 25–28 周

- 初始化、Swap Chain、命令队列
    
- PSO（Pipeline State Object）
    

### 第 29–32 周

- Root Signature
    
- Descriptor Heap
    
- Constant Buffer / Vertex Buffer
    

### 第 33–36 周

- Shadow Map
    
- PBR
    
- 后处理
    

输出：

- D3D12 mini renderer（渲染一个带光照的模型）
    

---

# 🧭 第四阶段（第 37–44 周）——光线追踪（Ray Tracing）

> 未来渲染方向，值得提前布局。

## 📘 BattleFire 建议购买：

- **Ray Tracing 渲染**（便宜 + 高价值）
    

## 内容：

- Bvh 构建
    
- 光线求交
    
- 漫反射 / 镜面反射 / 折射
    
- 路径追踪基础
    

输出：

- 用 CPU 或 GPU 实现一个简单光追 Demo

# 🧭 第五阶段（第 45–52 周）——作品集（非常关键）

> 没作品=没能力。你必须做个人项目。

## 🎯 必做目标（至少一个）

1. **小型渲染器**（OpenGL/D3D12 都行）
    
2. **Mini Game Engine**（含相机、模型加载、光照）
    
3. **光线追踪 Demo**
    

### 建议内容：

- Model → Shader → Light → Shadow → PBR → Post Process
    

输出（非常关键）：

- GitHub 项目
    
- 演示视频
    
- 说明文档
